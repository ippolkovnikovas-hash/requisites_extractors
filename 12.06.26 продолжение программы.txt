Что менять
Смысл такой:

не трогаем удачный happy-path для DOCX/PDF;

добавляем усиление именно для image/table-сценария;

app/services/pipeline_service.py
Это главный узел, сюда и надо вшить новый порядок. Сейчас для пользователя важно не ломать уже рабочий путь DOCX/PDF.

Новый flow:

extraction

normalization

LLM extraction

parse response

Псевдокод:

python
payload = extract_text(...)
normalized = normalize_payload(payload)

prompt_profile = "image_table" if payload.source_format == "image_table" else "text"
llm_raw = llm_client.extract(
    normalized.text,
    prompt_profile=prompt_profile,
    lines=normalized.lines,
)

parsed = parse_llm_response(llm_raw)

fallback = extract_fallback_fields(normalized.text)
merged = merge_llm_and_fallback(parsed, fallback)

validated = validate_requisites(
    merged,
    source_text=normalized.text,
    source_format=payload.source_format,
)
Логика merge:

если LLM поле пустое → берем regex;

если LLM поле невалидное → берем regex;

если оба есть и конфликтуют → сохраняем лучший по validator + warning.
